# Copilot Instructions for this Repository

En este proyecto, sigue siempre estos principios para código y pruebas:

- **Clean Code**:
  - escribe código claro, legible y mantenible
  - nombra variables/funciones con significado
  - evita complejidad innecesaria
  - divide en funciones/módulos con responsabilidad única

- **KISS (Keep It Simple, Stupid)**:
  - prioriza soluciones simples y directas
  - evita sobreingeniería y abstracciones prematuras
  - prefiere caminos explícitos y fáciles de entender

- **DRY (Don’t Repeat Yourself)**:
  - elimina duplicación de lógica
  - extrae componentes reutilizables y funciones comunes
  - centraliza reglas de negocio y validaciones en un solo lugar

- **Testing**:
  - escribe pruebas automatizadas (unitarias y de integración cuando aplique)
  - cubre casos felices y casos límite / de error
  - ejecuta las pruebas en cada cambio relevante
  - usa mocks/fakes según convenga para aislar unidades de código

> En general, haz cambios incrementales, revisables y con pruebas para validar comportamiento.
